Trabalho de Conclusão de Curso (TCC)

Segurança no Código Gerado por Modelos de Linguagem: Uma Revisão Bibliográfica de Vulnerabilidades e Abordagens de Defesa

Autores: Juan Tavares | Período: 8º Semestre

Resumo Executivo

Os Grandes Modelos de Linguagem (LLMs), sistemas de inteligência artificial treinados em vastos conjuntos de dados, tornaram-se ferramentas transformadoras na geração de código-fonte a partir de linguagem natural. No entanto, esses modelos geram código com vulnerabilidades devido ao treinamento em dados públicos que frequentemente contêm padrões inseguros.

Este estudo adotou uma Revisão Bibliográfica da Literatura, conduzida na base de dados Google Scholar, para mapear evidências recentes (a partir de 2023) sobre vulnerabilidades de segurança em código gerado por LLMs. Dos 42 artigos identificados inicialmente, 18 atenderam aos critérios de seleção e foram submetidos à análise detalhada.

Principais Descobertas

Vulnerabilidades Identificadas

  • 🔴 Injeção de SQL
  • 🟡 Injeção de Comando
  • 🟡 Gestão de Memória
  • 🔴 Cross-Site Scripting (XSS)
  • 🟡 Desserialização Insegura

Estratégias de Mitigação

  • ✓ Fine-tuning Especializado
  • ✓ Verificações de Código Estático
  • ✓ Análise Dinâmica
  • ✓ Testes de Segurança
  • ✓ Revisão de Código Humana

Contribuições da Pesquisa

  • Mapeamento Sistemático: Identificação das falhas de segurança mais comuns em LLMs
  • Análise Comparativa: Avaliação de diferentes abordagens de mitigação
  • Recomendações Práticas: Diretrizes para desenvolvedores usando LLMs
  • Roadmap de Pesquisa: Identificação de lacunas e oportunidades futuras

Conclusões

A pesquisa conclui que a adoção de mecanismos de verificação contribui decisivamente para um uso confiável de LLMs na geração de código. As vulnerabilidades não são inerentes aos LLMs, mas resultam de uma falta de processos de garantia de qualidade robustos. A combinação de fine-tuning especializado, análise automática e revisão humana fornece uma defesa em profundidade eficaz.
